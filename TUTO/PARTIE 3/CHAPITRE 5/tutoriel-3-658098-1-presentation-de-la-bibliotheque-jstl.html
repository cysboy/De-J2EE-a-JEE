Présentation de la bibliothèque JSTL

Le tutoriel que vous êtes en train de lire est en bêta-test. Son auteur souhaite que vous lui fassiez part de vos commentaires pour l'aider à l'améliorer avant sa publication officielle. Notez que le contenu n'a pas été validé par l'équipe éditoriale du Site du Zéro.
Par cysboy
Difficulté : Facile
Dans les précédents chapitres, nous avons pu voir comment utiliser :
  • les actions;
  • les scriplets;
  • les directives;
  • le langage d'expression (EL);


Même si les actions ne sont plus trop utilisées, il était important de les voir afin que vous ne soyez pas pris au dépourvu lorsque la bise fût venue si vous tombez dessus un jour...
Je vous avais aussi sensibilisé sur le fait que les scriplets sont le mal absolu et qu'il faut éviter de s'en servir. Non, il ne faut pas vous en servir mais, tout comme les actions, vous pouvez avoir à travailler sur un projet construit avec des scriptlets.

Il est maintenant temps de voir comment combler les lacunes de l'EL car, vous en conviendrez, l'EL manque cruellement de fonctionnalités : impossible de faire une boucle en EL alors que c'est une instruction indispensable !
C'est ce genre de fonctionnalité qu'offre les bibliothèques JSTL, alors, avant de d'apprendre à les utiliser, laissez-moi vous les présenter.

JSTL kesako ?

JSTL est l'acronyme pour Java Standard Tag Library. Il s'agit donc d'une bibliothèque qui incorpore toute une panoplie de tags (balises).
Au tout début de J2EE, cette dernière n'existait pas mais, au fil du temps, les responsables et développeurs de la plateforme se sont rendu compte que les utilisateurs développaient encore et encore les mêmes fonctionnalités afin de répondre à leurs besoins.

De ce fait, JSTL est né pour le plus grand plaisir des développeurs. Nous pourrons donc trouver tout un tas de fonctionnalités dans ces bibliothèques, à savoir :
  • faire des conditions;
  • faire des boucles;
  • faire des redirections;
  • faire des requête SQL;
  • traiter des fichiers XML;
  • ...


Les balises sont au format XML, tout comme les actions que nous avons déjà vues. Voici à quoi peut ressembler une instruction JSTL :

JSTL est donc un ensemble de balises utilisables afin de simplifier le travail de la personne en charge de la couche de présentation (vue dans le modèle MVC) que ce soit des développeurs et/ou WebDesigners.

Pourquoi ?

Le but premier était de combler un manque de fonctionnalités, comme je vous le disais précédemment mais, de nos jours, il est de proscrire l'utilisation de scriplets dans la couche de présentation ceci afin de respecter le modèle MVC.
Voici le schéma que je vous avais fourni dans un précédent chapitre :



Dans ce dernier, vous pourrez constater que la vue n'a pour rôle que d'afficher les données provenant du contrôleur ou du modèle. La vue ne sert qu'à ça ! D'ailleurs, les dernières recommandations d'Oracle, en terme de standard de programmation, précise qu'il faut proscrire au maximum les scriptlets. Après, vous êtes libre, mais les standards servent à définir une façon de faire, commune à toute personne dans le monde, ainsi, si vous respecter les standards, vous pourrez travailler aisément avec une autre personne (à l'autre bout du monde ?) qui les respecte aussi...

Ce n'est pas non plus la seule raison, JSTL permet d'avoir un code beaucoup plus propre qu'avec des scriptlets et d'avoir un langage commun entre plusieurs intervenant : Je reprend l'exemple du WebDesigner, as de la programmation HTML5 et CSS3. Ce dernier ne saura pas forcément coder en Java et apprendre un langage de programmation n'est pas du tout aussi simple que de savoir utiliser l'EL ou les actions...

De plus, toutes les instructions sont standardisées, ce qui fait que JSTL est complètement réutilisable !
Bon, je ne vais pas tergiverser sur ce sujet : c'est un point primordial du développement JEE.

Comment ?

Même s'il s'agit d'un standard, JSTL ne fait pas parti intégrante de Tomcat donc, même si je vous apprenais à l'utilisant maintenant, Tomcat ne saurait pas interpréter ces instructions.

Pour palier à ce problème, nous avons une petite manipulation à faire...
La première sera de télécharger une archive .jar qui contient les définitions JSTL. Je vous invite donc à la télécharger à partir de ce lien. Enregistrez cette archive dans le dossier "/lib" de Tomcat, ainsi, lorsque vous créerez un nouveau projet avec le runtime Tomcat, cette nouvelle archive sera automatiquement intégrée dans votre projet !

Par contre, pour que l'ajout de cette archive soit effective du point de vue de Tomcat, vous devez arrêter et redémarrer ce dernier afin qu'il recharge sa configuration et qu'il relise son dossier "/lib" afin d'intégrer la nouvelle archive.


Je tiens toutefois à vous précisez que, dans les prochaines parties de ce tutoriel, nous ne travaillerons plus avec Tomcat mais avec le serveur d'application Glassfish V3 qui, lui, embarque tout ce qui est nécessaire au développement JEE5 et JEE6, donc JSTL sera déjà présent mais bien d'autres choses encore.

Je vous invite maintenant à créer un nouveau projet, avec le runtime qui convient, afin de pouvoir commencer les hostilités.

Contenu

Avant de rentrer dans le vif du sujet, voici une présentation succincte de JSTL, son contenu et ses fonctionnalités.
Je vous invite maintenant à aller dans le dossier "/lib" du dossier d'installation de Tomcat, trouvez l'archive que nous venons d'y ajouter puis ouvrez-là avec votre archiveur préféré (Winzip, Winrar, 7zip...).

Voici ce que vous devriez obtenir :



Vous constatez qu'il y a deux packages et un dossier WEB-INF. Explorez ce dernier et regardez attentivement son contenu :



Oh ! des fichiers .tld !


Oui.
JSTL se construit de la même manière que les méthodes pour l'EL... Donc, si vous avez un bon esprit de déduction, vous devez donc avoir compris qu'il y a plusieurs bibliothèques et qu'elles se nomment :

Nom de la bibliothèque Raccourcis Description
Core c Contient toutes les actions de bases : condition, boucle...
Format fmt Utilisé pour le formatage des données (date française, américaine...).
XML x Permet de le traitement de fichier XML depuis une JSP.
SQL sql Permet d'accéder à une base de données depuis une JSP.
Function fn Offre toute une panoplie de méthodes à utiliser (contains, startsWith...)


Les plus aguerris d'entre vous aurons sans doute compris que nous pourrons créer nos propres balises JSTL... Il suffira de procéder comme la création de méthode pour l'EL : une classe, une fichier .tld et zou !
